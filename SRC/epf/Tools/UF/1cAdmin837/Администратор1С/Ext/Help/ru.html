Администратор 1С (© StepByStep )

Нестандартное редактирование реквизитов и табличных частей, следующих типов ссылочных объектов:

Справочники, Документы, ПланыОбмена, ПланыВидовХарактеристик, Бизнес-процессы, Задачи, ПланыСчетов, ПланыВидовРасчета.

Импорт 8.Х (базовый функционал): Справочники, ПланыВидовХарактеристик, Документы.

Множество дополнительных обработок.

Основные возможности (Обычная и Управляемая формы):

  • Редактировать реквизиты и табличные части объектов, в т.ч. отсутствующие в формах объекта типовой конфигурации.

  • Просмотреть движения документа по регистрам. Флажок "Показать движения".
    • Оптимизация просмотра движений документа: Установить Флажок "Скрывать пустые таблицы движений регистров".

  • Произвести проведение/отмену проведения документа. Основное меню "Проведение".

  • Произвести пакетное проведение: Документ-Основание + Текущий Документ. Основное меню "Проведение".

  • Просмотреть структуру подчиненности документа. Основное меню "Опции".

  • Создать документ копированием текущего. Основное меню "Создать копированием".

  • Просмотреть даты запрета корректировки/загрузки данных. Основное меню "Опции".

  • Произвести Просмотр/Регистрацию/Удаление регистрации изменений объекта в Планах обмена.

    • Регистрация изменений объекта в планах обмена включает и регистрацию "связанных" объектов - содержимое реквизитов объекта ссылочного типа.

    • Из регистрации исключаются объекты, не входящие в состав Плана обмена.

  • Просмотреть список регистров, в которых может встретиться объект (Регистратор / Измерение / Ресурс / Реквизит).

    • Вкладка "Метаданные регистров" для Обычной формы.

    • Вкладка "Дополнительно - Регистры" для Управляемой формы.

  • Стандартные действия с объектом. Основное меню "Записать" и "Удаление":

    • Подменю "Удаление": Пометить на удаление, Отменить пометку на удаление, Удалить непосредственно.
    • После записи документа необходимо произвести проведение документа, если осуществляется запись проведенного документа.

    • Флажок "Отключить контроль записи (триггеры записи)".

      • Флажок установлен: При выполнении записи или удаления данных будет производиться минимум проверок. так как при этом делается предположение, что производится запись данных, полученных через механизмы обмена данными, и эти данные корректны.

      • Триггер (Trigger) - это особый вид хранимых процедур, автоматически выполняемых при исполнении оператора Update, Insert или Delete над данными таблицы.

  • При редактировании реквизитов и табличных частей объектов анализируются типы и назначение следующих сущностей:

    • Тип "ХранилищеЗначения" - значение оставить в неизмененным;

    • "НомерПринятого", "НомерОтправленного" для предопределенного ПланаОбмена (с точки зрения механизма планов обмена) – значение оставить в неизмененным;

    • Признак "ЭтоГруппа" для справочника - значение оставить в неизмененным;

    • Тип "Строка" 50 и более символов – редактируется в окне для ввода многострочного текста;

    • Тип "Строка" неограниченная с именем реквизита, содержащим слова "АЛГОРИТМ" или "ЗАПРОС" - редактируются в форме для ввода многострочного текста с подсветкой синтаксиса в обычной форме;

    • Реквизиты и табличные части справочников, не соответствующие предназначению "ДляЭлемента" и/или "ДляГруппы", не выводятся на экран для выбранного объекта.

    • Вкладки с таблицами движений документов выводятся только для тех видов регистра, в которых документ может осуществлять движение.

      • Флажок "Скрывать пустые таблицы движений регистров" сокращает количество вкаладок, оставляя только непустые таблицы.

  • Просмотреть Параметры сеанса запуска 1С:Предприятие. Основное меню "Настройка".

  • Просмотреть и изменить Режимы функционирования 1С:Предприятие, Обработки. Основное меню "Настройка":

    • Установить/Отменить "Монопольный режим".

      • Необходимо для:

        • "Установки/Отключения ГлавногоУзла";

      • Рекомендуется для:

        • Групповые обработки справочников;

        •  "Поиск и замена ссылок";

        • "Перенумерация документов".

    • Установить/Отменить "Контроль записи (триггеры записи)" (свойство "ОбменДанными.Загрузка"). Если установлено, то при выполнении записи или удаления данных будет производиться минимум проверок.

    • Установить/Отменить "Показывать сообщения". Режим вывода дополнительных поясняющих сообщений о производимых действиях.

    • Установить/Отменить "Скрывать пустые таблицы движений регистров". Оптимизация работы обработки, в отдельных случаях в N-раз быстрее с активным флажком, чем с неактивным.

  • Просмотреть Карту маршрута бизнес-процесса. Основное меню "Опции".

  • Узнать ID объекта. Строка "Статус". Необходимо для Восстановления "Битой" ссылки.

  • Узнать Дату и Время создания объекта. Строка "Статус".

    • Необходимо для Восстановления "Битой" ссылки.

    • Для предопределенных элементов - пустая дата.

  • Узнать Имя предопределенного элемента, заданного в конфигураторе.

Вкладка "Дополнительно":

  • Рекомендуемые параметры для выполнения групповых обработок:
    • Установить Флажок "Выполнять в транзакции". Установлен по умолчанию.
    • Установить Флажок "Монопольный режим". По умолчанию не установлен.

Дополнительные возможности (Управляемая форма):

  • Импорт 8.Х. БАЗОВЫЙ ФУНКЦИОНАЛ.
    • Импортировать объекты ссылочного типа (справочники, планы видов характеристик, документы) из "Архивной" или "Сторонней" информационной базы 1С:Предприятие 8.Х.
    • Подробности - FAQ в форме обработки.

  • Объекты ссылочного типа (Одиночные и Групповые обработки):

    • Просмотреть ссылки на выбранный объект в других объектах информационной базы. Вкладка "Дополнительно - Ссылки".

    • Произвести замену ссылки выбранного объекта на другой объект информационной базы. Вкладка "Дополнительно - Ссылки".

      • Исключение: ПланыОбмена. Заблокировано, т.к. это нецелесообразно.

      • Рекомендуется использовать Флажок "Одинаковый тип объектов при замене", накладывающий ограничения на тип выбираемого заменяющего объекта в соответствии с типом выбранного объекта.

    • Поиск и Восстановление "Битой" ссылки (<Объект не найден>).
      • При определении Уникального идентификатора определяется и дата создания Объекта Информационной Базы. Эта дата определяет дату архивной копии, начиная с которой можно восстановить содержаниеобъекта.
      • Процедура Восстановления подробно описана непосредственно в форме Обработки.

  • Документы (Групповые обработки):

    • Произвести перенумерацию документов. Вкладка "Дополнительно - Документ - Перенумерация".

      • Перенумерация документов возможна за произвольный период.

      • Рекомендуется осуществлять перенумерацию по организации, префиксу в соответствии с периодичностью нумерации по типу документа.
        • Документы с похожими префиксами можно отфильтровать следующими способами:

          • По Организации, если такой реквизит присутствует в документе.

          • По ПрефиксуИсключение, содержащий исключаемые префиксы, разделенные  ";" (точка с запятой).

            • ВАЖНО. В ПрефиксеИсключение недопустимы пробелы, если они не являются частью префикса. Пробелы считаются значащими символами.

      • Принцип работы:

        • Получить начальные параметры для перенумерации документов Кнопка "Получить начальные данные с соответствии с периодичность документа".

        • Выбрать Организацию, если у документа есть соответствующий реквизит.

        • Произвести поиск в соответствии с указанными параметрами.

          • В случае, если реквизит Организация отсутствует у объекта информационной базы и встретились документы, относящиеся к разным организациям, но с похожими префиксами, то можно воспользоваться корректировкой реквизита "ПрефиксИсключение" для фильтрации документов с такими номерами.

        • Выбрать Необходимые документы Флажки "+/-" (реквизит "Включено").

        • Указать НовыйПрефикс и  НачальныйНомер.

        • Кнопка "Перенумеровать документы" завершает процесс перенумерации.

    • Произвести проведение документов. Вкладка "Дополнительно - Документ - Проведение".

      • Проведение документов возможно за произвольный период.

      • Принцип работы:

        • Установить начальные параметры для проведения документов.

          • При первоначальном формировании списка Виды документов, исключаются виды документов, у которых "Проведение" неразрешено (например, документ КорректировкаРегистров).

          • При выборе параметра "Организация" будет произведен отбор видов документов, содержащих этот реквизит.

          • Параметр "Ожидать снятие блокировки с документа" актуален при немонопольном проведении документов с ненулевой вероятностью, что какой-либо из проводимых документов в момен проведения окажется заблокированным другим пользователем.

          • Дополнительный параметр "Время ожидания снятия блокировки" может позволить пользователю в соответствующий интервал времени завершить свои действия с документом и закрыть документ.

        • Выбрать виды документов.

        • Выбрать "Проведенные/Непроведенные/Все".

        • Кнопка "Провести документы" завершает процесс перепроведения документов.
    • Произвести удаление документов. Вкладка "Дополнительно - Документ - Удаление".

      • ВАЖНО. Перед удалением документов произвести резервное копирование информационной базы.

      • ВАЖНО. Удаление документов производится вне транзакции.

      • Удаление документов возможно за произвольный период.

      • Принцип работы:

        • Установить начальные параметры для удаления документов.

          • При первоначальном формировании списка Виды документов, исключаются виды документов, у которых "Проведение" неразрешено (например, документ КорректировкаРегистров).

          • При выборе параметра "Организация" в колонке "О" (Организация) будут обозначены документы, содержащие реквизит "Организация".

          • При выборе параметра "Склад" в колонке "С" (Склад) будут обозначены документы, содержащие реквизит "Склад".

          • При выборе параметра "Магазин" в колонке "М" (Магазин) будут обозначены документы, содержащие реквизит "Магазин".

          • При выборе параметра "Касса" в колонке "К" (Касса) будут обозначены документы, содержащие реквизит "Касса".

          • При выборе параметра "КассаККМ" в колонке "ККМ" (КассаККМ) будут обозначены документы, содержащие реквизит "КассаККМ".

          • При выборе параметра "Подразделение" в колонке "ПД" (Подразделение) будут обозначены документы, содержащие реквизит "Подразделение".

          • При выборе параметра "ПодразделениеОрганизации" в колонке "ПДО" (ПодразделениеОрганизации) будут обозначены документы, содержащие реквизит "ПодразделениеОрганизации".

            Колонка "ПЧ" (возможное наличие подчиненных документов), т.е. документ может служить основанием для других документов.

          • При получении списка удаляемых документов, документы, имеющие соответствующие реквизиты, но с другими значениями - исключаются из списка удаляемых.

          • Выбрать, при необходимости, "в т.ч. подчиненные" документы.

          • Опция: Отбор документов по номенклатуре из выбранной группы справочника ("Номенклатура", если присутствует в конфигурации). Отбор документов по номенклатуре - длительная процедура. Рекомендуется после удаления документов по основному отбору. Возможно удаление номенклатуры, входящей в выбранную группу.

        • Выбрать действие над документами: "Пометка на удаление"/"Удаление".

          • Рекомендуется - "Пометка на удаление". Фактическое удаление документов произвести штатной процедурой удаления объектов из информационной базы.

          • При выборе "Удаление" будет произведено непосредственное удаление документов без возможности восстановления.

        • Кнопка "Список удаляемых".
        • ВАЖНО. Из списка удаляемых документов исключить документы, которые не следует удалять. Флажок "П" (Пометка).
        • Кнопка "Удалить документы".
        • При указании в настройке обработки "Показывать сообщения" будут выведеные пояснения по исключаемым из списка удаляемых документов.
    • Получить статистику документов. Вкладка "Дополнительно - Документ - Статистика".
  • Ошибки Информационной Базы:
    • Актуально для Распределенной Информационной Базы.
    • Получить список непроведенных документов с движениями. Вкладка "Дополнительно - Документ - Движения".
      • Удалить движения непроведенных документов, имеющих движения.
        • Исключение: документы типа "Корректировка регистров" и др.: проведение неразрешено, но документ может иметь движения.
    • Получить список проведенных документов без движений. Вкладка "Дополнительно - Документ - Движения". 
      • Провести "проведенные" документы, не имеющие движения.
    • Общее:

      • Принцип работы:

        • Установить начальные параметры отбора документов.

          • При первоначальном формировании списка Виды документов, исключаются виды документов, у которых "Проведение" неразрешено (например, документ КорректировкаРегистров).

          • При выборе параметра "Организация" в колонке "О" (Организация) будут обозначены документы, содержащие реквизит "Организация".

          • При получении списка удаляемых документов, документы, имеющие соответствующие реквизиты, но с другими значениями - исключаются из списка отобранных.
          • "Активные/Неактивные записи регистров".
        • Выполнить: "Найти непроведенные с движениями"/"Найти проведенные без движений".
        • Выполнить действие:
          • Для непроведенных с движениями: "Очистить движения документов".
          • Для проведенных без движений: "Провести документы".
  • Прочие возможности:
    • Просмотреть список активных пользователей. Основное меню "Сервис".
    • Просмотреть историю изменений объекта. Основное меню "Опции".
    • Просмотреть общий журнал регистрации. Основное меню "Сервис".
    • Просмотреть/Произвести регистрацию/Удалить регистрацию изменений множества объектов в планах обмена. Основное меню "Сервис".

Специфические возможности (Управляемая форма):

  • Справочники (Групповые обработки):

    • Принцип работы:

      • Необходимо выбрать Действие:

        • "ЧистыйСтроковыйРеквизит_БезЛишнихСимволов":

          • Реквизит может быть только типа "Строка".

          • "Чистый" реквизит: Из реквизита удаляются символы: Символы.НПП, Символы.ТАБ и др. аналогичные, а также лишние пробелы.

        • "УдалитьНедопустимыеСимволыXML":

          • Реквизит может быть только типа "Строка".

          • Из реквизита удаляются только недопустимые для XML символы. Могут оставаться нечитаемые символы ("абраказябры").

        • "ИсключитьНеЧитаемыеСимволыИзСтроки":

          • Реквизит может быть только типа "Строка".

          • Из реквизита исключаются символы не входящие в:

          • |Латиница = ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz;
            |Кирилица = АБВГДЕЁЖЗИЙКЛМНОПРСТУФХЦЧШЩЪЫЬЭЮЯабвгдеёжзийклмнопрстуфхцчшщъыьэюя;
            |Греческие = ΑΒΓΔΕΖΗΘΙΚΛΜΝΞΟΠΡΣΤΥΦΧΨΩαβγδεζηθικλμνξοπρστυφχψω;
            |Цифры = 0123456789;
            |СпециальныеСимволы = ~`!@#$%^&*(){}[]_-=+\|/*:;.<>?,№«» ;
            |ДвойнаяКавычка = """";
            |ОдинарнаяКавычка = ';
            |АпострофОбратный = ́;  // КодСимвола 769. Обратный для символа на букве Ё.
            |АвторскоеПраво = ©;  // КодСимвола 169. Copyright - латинская буква C в окружности - авторское право.
            |Зарезервировано = ®;   // КодСимвола 174. Registered - латинская буква R в окружности - товарный знак.
            |ТоварныйЗнак = ™;   // Верхний  индекс ТМ.
            |ШирокоеТире = —;   // КодСимвола 8212.

        • "Реквизит_Значение":

          • Реквизит может быть произвольного типа из списка реквизитов, возможных для вида выбранного объекта информационной базы.

          • Реквизиту будет присвоено Значение в соответствии с возможными типами этого реквизита.

          • ВАЖНО. При выборе реквизита производится проверка "Блокированный-Неблокированный", с выдачей предупреждения - если блокированный реквизит.

        • "НаименованиеПолное_НаименованиеИОписание":

          • Реквизиту "НаименованиеПолное" (если такой существует у выбранного вида объекта) будет присвоено значение составленное из 2-х реквизитов "Наименование" и "Описание".

          • Если реквизит "Описание" пустое, то будет присвоено значение реквизита "Наименование".

        • "Наименование_АртикулВНаименование":

          • Реквизиту "Наименование" (если такой существует у выбранного вида объекта) будет присвоено значение составленное из 2-х реквизитов "Артикул" и "Наименование".

          • Если реквизит "Артикул" пустое, то изменение реквизита не произойдет.

          • Если Артикул 1-ым (первым) словом присутствует, то отказаться от преобразования.

          • Если Артикул присутствует в наименовании несколько раз, то остается только 1-ым (первым) словом.

        • "Наименование_НаименованиеБезАртикула" (обратное действие):

          • Реквизит "Наименование" (если такой существует у выбранного вида объекта) очищается от присутствия в нем значения реквизита "Артикул".

          • Если реквизит "Артикул" пустое, то изменение реквизита не произойдет.

        • "Наименования_ПоОКОПФ" (Стандартизация содержания полей наименований с применением наиболее употребимой части ОКОПФ):

          • Реквизиты "Наименование", "НаименованиеПолное", "НаименованиеСокращенное" (если такие существуют у выбранного вида объекта) реорганизуются в соответствии с ОКОФ (Общероссийский классификатор основных фондов).

          • Обработка имеет смысл для справочников "Контрагенты" и "Организации".

            • Принцип действия:

              • Ключевое значение - реквизит "Наименование".

                  • Из Наименования, например, "ООО БеломорКанал" извлекается 1-ое (первое) слово "ООО".

                  • Если 1-ым словом является аббревиатура из ОКОПФ, то элемент обрабатывается.

                  • В результате обработки должны получиться значения реквизитов подобно:

                    • Наименование - БеломорКанал ООО

                    • НаименованиеСокращенное - ООО БеломорКанал

                    • НаименованиеПолное - Общество с ограниченной ответственностью БеломорКанал.

        • "НеиспользуемыеЭлементы_ПоискИУдаление" (Пометка на удаление неиспользуемых элементов справочника):

          • Поиск производится по ссылкам.

          • Возможна фильтрация по дате создания элемента справочника.

          • В поле "Реквизит" записывается "Наименование".

          • Фактическое удаление производится штатными средствами 1С:Предприятие:

            • Обычное приложение: Главное меню - Операции - Удаление помеченных объектов.

            • Управляемое приложение: Администрирование - Удаление помеченных объектов.

            • Фактическое удаление рекомендуется производить в монопольном режиме.

          • ВАЖНО. Помеченные на удаление элементы раньше (до выполняемой данного действия) исключаются из поиска и последующей пометки на удаление.

          • ВАЖНО. Предопределенные элементы исключаются из поиска и последующей пометки на удаление (удалить предопределенные элементы в режиме 1С:Предприятие невозможно).

          • ВАЖНО. В поле "Объект/Группа" рекомендуется выбирать Группу/Элемент-Родитель для иерархического справочника. Обработка всего справочника может быть очень длительной, исключение неиерархический справочник (предположение: неиерархические справочники - небольшие по количеству элементоа).

      • Необходимо выбрать Область Действия: Элемент/Группа/ВесьСправочник в реквизите формы.

      • Необходимо выбрать обрабатываемый Реквизит из списка реквизитов, возможных для вида выбранного объекта информационной базы.

      • Необходимо выбрать Значение Реквизита:

        • Для действия "ЧистыйСтроковыйРеквизит_БезЛишнихСимволов" тип "Строка".

        • Для действия "Реквизит_Значение" тип в соответствии с возможными типами этого реквизита.

      • Необходимо произвести Поиск в соответствии с областью действия: Элемент/Группа/ВесьСправочник.

      • Необходимо выбрать Необходимые элементы справчника Флажки "+/-" (реквизит "Включено").
